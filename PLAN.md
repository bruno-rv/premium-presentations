# Plan: v2.1.0 — Tier 2 budget loop, timed teleprompter, /present-architecture + /present-postmortem

_Locked via grill-with-docs — by Claude + Bruno. Terms per CONTEXT.md. Revised after Codex rounds 1–4; APPROVED round 5 (see PLAN-REVIEW-LOG.md)._

## Goal

Close the rehearsal-coach loop deferred from v1.3: decks carry per-slide **Slide Budgets** authored in the Slide Map, the presenter popup reports **delta vs plan** (non-uniform), and the teleprompter gains **timed scroll** driven by those budgets. Alongside, ship two new **Recipes** — `/present-architecture` (live codebase scan) and `/present-postmortem` (user-supplied incident doc) — modeled on `/present-pr`. One release: v2.1.0.

## Canonical rules (apply everywhere below)

- **Slide identity is the stable Slide Map `ID`**, matched to section DOM ids (as `partial_regen.py` already does). Ordinal `#` and Title are display/diagnostics only.
- **Time grammar:** `Budget (ms)` is authoritative — a decimal integer, no signs/whitespace, **min 1,000** (sub-second budgets rejected), max 7,200,000 (2 h/slide), within JS safe-integer range. `Budget (mm:ss)` is derived display matching `^\d{2,}:[0-5]\d$` and MUST equal `floor(ms/1000)` rendered as zero-padded minutes:seconds.
- **Serializer ownership:** `slide_spec.py` owns the Python parse/format implementation; `premium-presenter.js` has the single JS counterpart. A canonical **JSON test-vector file** (valid + invalid cases) is consumed identically by pytest and the node tests — both implementations must agree on every vector.
- **Budget column states (exactly three):** (a) headers absent, or headers present with ALL budget cells empty → **budgetless deck**, no gate, no runtime budgets; (b) headers present and EVERY row populated with valid values → **budgeted deck**; (c) anything else (one header without the other, any populated subset, any invalid value) → **validation failure**.
- **Runtime acceptance:** one `readSlideBudgets()` operates on the normalized DOM — accepts only a complete valid vector (every section carries a valid in-range `data-budget`, unique slide identities) — else atomically falls back to vs-average/manual-scroll behavior with one console diagnostic. (Duplicate source attributes are invisible post-parse; catching them is the validator's job, see step 3.)
- **Comparison label:** the UI term is exactly **"vs average"** (fallback) or **"vs plan"** (budgeted) — the current `"vs avg"` string is replaced everywhere (UI, APIs, docs, tests) per CONTEXT.md.

## Approach

### Workstream A — Tier 2 Slide Budget loop

1. `references/slide-spec-template.md`: add optional Slide Map columns `Budget (mm:ss)` + `Budget (ms)`. Document the three column states, atomic header pair, time grammar, and the Slide-Budget / Color-semantics-budget distinction.
2. `scripts/slide_spec.py`: parse the budget columns (ID-keyed) per the canonical rules, exposing them to validators and tooling. **HTML emission of `data-budget` is owned by the agent HTML-emit contract** (SKILL.md / references generation rules: set `data-budget="<ms>"` on each slide `<section>`, verbatim from spec) — `spec_generator.py` only gains template scaffolding for the new columns.
3. `scripts/validate_deck.py` (via `deck_doctor.py`) hard gate: three-state column rule; time grammar (bounds, integer, mm:ss ↔ ms agreement). **HTML-side checks run unconditionally** (spec argument is optional): attribute completeness across sections, grammar/bounds of every `data-budget`, duplicate source attributes (raw-markup scan — the DOM normalizes duplicates away), orphan attributes, and — whenever any `data-budget` is present — **nonempty, unique section IDs** (so the doctor never passes an artifact the popup's `readSlideBudgets()` would reject). **Spec ↔ HTML ID parity added only when a spec is supplied.** Non-zero exit on violation. Budgetless decks: zero false positives.
4. `assets/shared/premium-presenter.js`: `readSlideBudgets()` + one centralized "planned-time vector + comparison-label" helper. ALL consumers route through it: timeline deltas, status line (currently claims uniform "plan …/slide"), `getLastRunDeltas()`, labels ("vs plan" / "vs average" per canonical rules).
5. Rehearsal suggested-budget export: when the deck has stable section IDs, an **ID-keyed** table `| ID | Budget (mm:ss) | Budget (ms) |` from the median vector, cells Markdown-escaped (`|`, backslash, newlines). **Sample eligibility:** a slide's median counts only if it has at least one in-range observation (1,000 ms – cap) across saved runs; if ANY slide lacks one (unvisited, zero, sub-second, over-cap), the export refuses with an explanatory message naming the offending slides — no silent clamping, no partial suggestion. **Legacy decks without stable IDs:** export falls back to ordinal + title with an explicit "initialize stable IDs in the Slide Map before merging" notice — never emits blank identifiers. Loop closure is an **agent-mediated merge** by ID, verified by the deck_doctor gate — no blind-paste claim. JSON export is **versioned with an explicit identity mode**: `{v: 2, identity: "id" | "ordinal", rows: [...]}` where each row carries either the stable ID or ordinal + title — never fabricated blank IDs; same serializer.
6. Partial regeneration — transactional design with a **public CLI contract**: `plan --json` **keeps `changed` as the backward-compatible union** and adds `schemaVersion` plus two partitions `contentChanged` and `budgetOnly` (existing key-set tests and docs — README, SKILL.md, runtime.md — updated in the same change); `apply` accepts **zero `--fragment` args when all changes are budget-only** and requires exact fragment coverage of `contentChanged` rows otherwise. One transaction applies budget-attribute sync (attribute update, body untouched) for budget-only rows and replacement fragments for content rows, then updates ALL affected embedded state (`premium-regen-state` semantic rows, row hashes, section hashes) and asserts the resulting baseline is clean (a follow-up no-op plan reports zero drift). `slide_html.py:validate_fragment()` gains `data-budget` parity enforcement for replacement fragments.

### Workstream B — Timed teleprompter scroll (depends on A)

7. Engage rule: deck passes `readSlideBudgets()` → `p` starts **timed scroll**; otherwise today's manual constant px/s, untouched.
8. Progress model: `progress = accumulatedProgress + (performance.now() − epoch) × multiplier / budgetMs`, clamped [0, 1]; `scrollTop = progress × (scrollHeight − clientHeight)`. **On any multiplier change, accumulate progress and rebase the epoch** so position stays continuous. Target distance remeasured after notes render, teleprompter-mode toggle, glossary/font settle, and (throttled) popup resize. Notes that don't overflow → no motion.
9. State machine (explicit): `p` sets **play intent**. While intent is on, each slide change resets scrollTop to top (DOM replacement already does), zeroes accumulated progress, and starts a fresh epoch — motion continues without a new keypress, matching today's manual-mode behavior. **Pause (`p` again): commit current progress to accumulatedProgress, clear only the epoch — position holds. Resume (`p`): keep accumulatedProgress, set a fresh epoch — scroll continues from where it stopped, no restart, no backward jump.** Popup close/reopen clears intent and progress. **Reduced-motion rule, stated exactly:** users with `prefers-reduced-motion: reduce` MAY start scrolling with an explicit `p`; no load, mode toggle, slide change, or state restoration may ever create play intent by itself. Tested as such.
10. `[` / `]`: in timed mode a **speed multiplier** stored in integer tenths (10 = ×1.0), clamped [5, 20] (×0.5–×2.0); in manual mode today's px/s nudge, unchanged.
11. Persistence migration: `localStorage['premium-teleprompter']` is currently a **plain numeric string** (manual rate). New versioned schema `{v: 2, manualRate: number, multiplierTenths: number}`; reader accepts the legacy numeric string and migrates on first write. No saved-rate loss.

Fixed arc: timeline → impact → root cause → remediation → lessons, blameless tone; sections without evidence render "not documented". **PII minimization by default — enumerated classes:** person names, email addresses, phone numbers, IP addresses, and customer/tenant/account identifiers are pseudonymized (roles like "on-call engineer", labels like "Customer A"); retaining any direct identifier requires explicit user opt-in; credential redaction as in 13; deck marked internal-audience by default.

## Key decisions & tradeoffs

- **All-or-nothing budgets with exactly three column states, enforced at validator AND runtime** — portable HTML can be hand-edited, so runtime acceptance is atomic.
- **`Budget (ms)` authoritative (min 1 s), mm:ss derived display; one Python owner (`slide_spec.py`) + one JS counterpart, held together by shared JSON test vectors.**
- **`data-budget` emission owned by the agent HTML-emit contract, gated by validator parity.**
- **Slide identity = stable `ID`; legacy no-ID decks get a labeled ordinal/title export, never blank IDs.**
- **Timed scroll: play intent set only by explicit `p` (reduced-motion users included); epoch-based progress with multiplier rebase — continuous position, no drift.**
- **Recipes: all source collection/caps/redaction centralized in deterministic `recipe_source_guard.py` (offline-testable); architecture = tracked-file scan; postmortem = validated user file with explicit argument contract and PII minimization; injection resistance = directives-not-followed, evaluated manually per provider; commands Claude-only, Codex routes via skill.**
- **One release (v2.1.0).**

## Risks / open questions

- Multiplier persistence is global (matches today's global rate), not per-deck — accepted.
- Resize remeasure throttled to animation-frame granularity — accepted staleness.
- Credential-pattern and PII redaction are **best-effort pattern matching**: only explicitly recognizable patterns are guaranteed; arbitrary unlabeled names/account identifiers may pass through. Ambiguous identities require manual review or an explicit user-supplied pseudonymization map — documented limitation in both recipe briefs.

## Out of scope

- Elapsed-time syncing of scroll to slide-entry timestamps (no catch-up jumping mid-slide).
- Site-scrape/logo brand input; automated deck-diff script; LAN follow-along changes.
- Any shared-runtime change beyond the `premium-presenter.js` sections named above.
